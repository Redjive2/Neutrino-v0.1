The page is a single-page application shell.  Nothing is server-rendered; all
dynamic content is injected by app.js at runtime.

────────────────────────────────────────────────────────────────────────────────
MODULE-LEVEL STATE
────────────────────────────────────────────────────────────────────────────────

  session            { username, token: number[32], profilePic? }
                     Persisted in sessionStorage as JSON.
                     token is the server's rolling [32]byte auth token,
                     stored as an array of 32 ints (Go JSON encoding).

  apiQueue           Promise chain — all authenticated POST requests are
                     serialised through this to prevent token desync.
                     (Server rotates token on every authenticated response.)

  profilePicCache    Map<username, mediaId|null>
                     Populated lazily by ensureProfilePics().  Never
                     invalidated during a session.

  pendingAttachments Array<{id, mimeType, name, objectURL}>
                     Files the user has selected but not yet sent.
                     objectURL is a Blob URL for thumbnail preview.
                     Cleared on send and on channel switch.

  currentServer      string | null
  currentCategory    string | null
  currentChannel     string | null
  lastSeenId         int — highest message ID rendered; -1 = channel empty
  renderedMessages   Array<{from, content, id, time, attachments[]}>
                     The flat in-memory list of displayed messages.
  hasMoreMessages    bool — whether a "Load older" page is available
  loadingMore        bool — guard against concurrent load-more requests
  navGeneration      int — incremented on every server/channel switch;
                     stale async responses are discarded by comparing gen.
  pollTimer          setInterval handle
  sessionCheckTimer  setInterval handle
  exploreOpen        bool

────────────────────────────────────────────────────────────────────────────────
KEY FUNCTIONS
────────────────────────────────────────────────────────────────────────────────

SESSION
  loadSession()          Restore session from sessionStorage on page load.
  saveSession()          Write session to sessionStorage.
  clearSession()         Null out and remove from storage.

API CLIENT
  apiPost(path, body)    Enqueues an authenticated JSON POST through apiQueue.
                         Always reads the current token at send time, not
                         at enqueue time.  Updates session.token on response.
  _apiPost(path, body)   Internal: executes the actual fetch.
  checkSessionAlive()    Token-free poll of /get/sessionstatus.  Used by the
                         heartbeat timer.  Also syncs session.profilePic.
  authBody(extra)        Returns { username, token, ...extra } from session.

MEDIA UPLOAD
  tokenToHex(token)      Converts [32]byte int array → 64-char hex string
                         for multipart form auth (upload endpoint requires it).
  apiUpload(file)        Enqueues a multipart POST /media/upload through
                         apiQueue (so token stays in sync).
  handleFilesSelected(files)
                         Uploads each selected file, appends to
                         pendingAttachments, calls renderAttachmentPreviews().
  renderAttachmentPreviews()
                         Rebuilds the #attachment-preview-strip DOM from
                         pendingAttachments.  Each tile has a thumbnail (or
                         a 🎬 icon for video) and a remove button.
  changeProfilePic()     Creates a temporary file input (images only),
                         uploads the selection, calls /edit/profilepic/{id},
                         updates session.profilePic and profilePicCache.

PROFILE PICTURE CACHE
  ensureProfilePics(usernames)
                         Fetches /get/sessionstatus for every username not
                         yet in profilePicCache (in parallel).  Called before
                         any render that needs avatar images.

AUTH
  login(username, password)    POST /open/session/{user}; saves token.
  register(username, password) POST /new/user/{name}.
  logout()                     POST /close/session; clears session + reloads.

DATA FETCHING
  fetchManifest()              GET all servers.
  fetchMyServers(manifest)     Filters manifest to servers the user is in.
  fetchServerData(serverName)  GET categories/members for one server.
  fetchChannelData(srv,cat,ch,before)
                               GET member list + paginated history.
                               before=0 → newest page; before=ID → older page.
  fetchNewMessages(srv,cat,ch,sinceId)
                               GET messages with id > sinceId (incremental poll).

RENDERING
  formatText(raw)        Minimal markdown: **bold**, *italic*, `code`.
                         HTML-escapes input first.
  buildAttachmentsHTML(attachments)
                         Returns HTML for inline images (<img class="message-img">)
                         and videos (<video class="message-video" controls>).
  buildMessageHTML(msgs) Renders a list of message objects to HTML strings.
                         Groups consecutive messages from the same author.
                         Uses profilePicCache for avatar images.
  rebuildDOM()           Replaces #messages-container innerHTML, preserves
                         scroll position if user was already at the bottom.
  renderMessages(history)   Replaces renderedMessages, calls rebuildDOM().
  appendMessages(newMsgs)   Appends to renderedMessages, calls rebuildDOM().
  renderMemberList(members) async — calls ensureProfilePics first, then
                            rebuilds #members-sidebar.
  renderServerList(servers) Rebuilds #server-icons-container.
  renderChannelSidebar(serverName, serverData)
                            Rebuilds #channel-list with categories and
                            channels; attaches click, add, and delete handlers.

CHANNEL / SERVER MANAGEMENT
  promptCreateCategory()   prompt() → POST /new/category
  promptCreateChannel(cat) prompt() → POST /new/channel
  removeCategory(cat)      POST /remove/category
  removeChannel(cat, ch)   POST /remove/channel
  deleteServer()           POST /remove/server; falls back to recoverFromServerLoss

MEMBER ACTIONS
  promptInviteUser()       Invites to server then to channel.
  kickUserFromChannel(u)   POST /kick/channel

USER ACCOUNT ACTIONS
  changeUsername()         prompt() → POST /edit/user/{newname}
  changePassword()         prompt() × 2 → POST /edit/password
  deleteAccount()          prompt() → POST /remove/user
  changeProfilePic()       (see Media Upload above)

NAVIGATION
  switchServer(name)   Stops poll, resets state, fetches server data,
                       renders sidebar, auto-selects first channel.
  switchChannel(cat, ch)
                       Stops poll; clears pendingAttachments; increments
                       navGeneration; fetches channel data; pre-fetches
                       profile pics for all members + history senders;
                       renders messages and member list; starts polling.

POLLING
  startPolling() / stopPolling()   setInterval(poll, 2000)
  startSessionCheck() / stopSessionCheck()   setInterval(sessionHeartbeat, 10000)
  sessionHeartbeat()   Calls checkSessionAlive(); force-logout if dead.
  poll()               If lastSeenId < 0: full channel fetch.
                       Otherwise: incremental GET /get/chat/…/{lastSeenId}.
                       Pre-fetches profile pics for new senders before render.

RECOVERY
  recoverFromChannelLoss()  Channel gone or access revoked; re-fetches server
                            data and selects first available channel.
  recoverFromServerLoss()   Server gone; re-fetches manifest and selects
                            first accessible server.
  resetChatArea(reason)     Clears chat header and messages area.

MESSAGE SENDING
  sendMessage(text)    Collects pendingAttachments IDs, clears pending,
                       POSTs /new/message with { content, attachments: [...] },
                       then triggers a poll to display the result immediately.
  loadMoreMessages()   Fetches the next older page using the oldest rendered
                       message ID as the `before` cursor.  Pre-fetches profile
                       pics, prepends older messages, restores scroll position.

UI HELPERS
  updateUserUI()       Updates #current-username and #user-avatar.
                       Shows <img> avatar if session.profilePic is set.
  initEmojiPicker()    Populates #emoji-grid with buttons from EMOJIS array.
  showLogin() / hideLogin()   Toggle .login-overlay.open
  closeEmojiPicker()   Removes .open from #emoji-picker.

INIT
  DOMContentLoaded handler
    — wires all static button event listeners (login, register, message
      input, attach button, emoji, delete message, member invite/kick,
      explore, toggle members, account menu)
    — if session exists: calls initApp(); else: showLogin()

  initApp()
    1. Validates session via GET /get/manifest (also rotates token once)
    2. updateUserUI() + seeds profilePicCache for self
    3. startSessionCheck()
    4. fetchMyServers() → renderServerList()
    5. switchServer(first server) or shows "No servers" message

────────────────────────────────────────────────────────────────────────────────
DATA FLOW SUMMARY
────────────────────────────────────────────────────────────────────────────────

  User navigates to channel
    switchChannel()
      → fetchChannelData()           (POST /get/channel/…)
      → ensureProfilePics()          (parallel /get/sessionstatus for each user)
      → renderMessages()             (fills renderedMessages, calls rebuildDOM)
      → renderMemberList()           (fills sidebar)
      → startPolling()

  Poll fires (every 2s)
    poll()
      → fetchNewMessages()           (POST /get/chat/…/{lastSeenId})
      → ensureProfilePics(new senders)
      → push to renderedMessages, rebuildDOM()

  User sends a message
    sendMessage()
      → apiPost(/new/message)        (includes attachments: [id, …])
      → poll()                       (immediately refresh)

  User attaches a file
    attach-btn click → file picker
    handleFilesSelected()
      → apiUpload(file)              (multipart POST /media/upload)
      → push to pendingAttachments
      → renderAttachmentPreviews()

  User sets profile picture
    changeProfilePic()
      → apiUpload(image)
      → apiPost(/edit/profilepic/{id})
      → session.profilePic = id
      → profilePicCache.set(username, id)
      → updateUserUI()

  Session heartbeat (every 5s)
    checkSessionAlive()
      → GET /get/sessionstatus       (token-free)
      → syncs session.profilePic / profilePicCache
      → force-logout if inactive

────────────────────────────────────────────────────────────────────────────────
API ENDPOINTS USED BY THE CLIENT
────────────────────────────────────────────────────────────────────────────────

AUTH / SESSION
  POST /new/user/{name}              Register
  POST /open/session/{user}          Login → returns token
  POST /close/session                Logout
  POST /get/sessionstatus            Heartbeat (token-free); returns active+profilePic

SERVERS
  POST /get/manifest                 List all servers (name, owner)
  POST /get/server/{server}          Server detail (owner, members, categories)
  POST /new/server/{name}
  POST /remove/server/{server}
  POST /edit/server/{server}/{newname}

CATEGORIES
  POST /new/category/{server}/{name}
  POST /remove/category/{server}/{category}

CHANNELS
  POST /new/channel/{server}/{category}/{name}
  POST /remove/channel/{server}/{category}/{channel}
  POST /edit/channel/{server}/{category}/{channel}/{newname}
  POST /get/channel/{server}/{category}/{channel}   Paginated history + members
  POST /get/chat/{server}/{category}/{channel}/{id} Incremental poll

MESSAGES
  POST /new/message/{server}/{category}/{channel}
  POST /remove/message/{server}/{category}/{channel}/{id}
  POST /edit/message/{server}/{category}/{channel}/{id}

MEMBERS
  POST /invite/server/{user}/{server}
  POST /invite/channel/{user}/{server}/{category}/{channel}
  POST /kick/channel/{user}/{server}/{category}/{channel}

USER
  POST /edit/user/{newname}
  POST /edit/password
  POST /remove/user

MEDIA
  POST /media/upload                 Multipart; auth via form fields (hex token)
  GET  /media/{id}                   Serve file (no auth)
  POST /edit/profilepic/{mediaid}    Set own profile picture
